For this project, I scoped toward a geographic journal app. Early on, I locked in the core concept: private and friends-only access (avoiding the need for moderation), GPS coordinates fixed at entry creation for data integrity, and optional photo-plus-text content so that simple "I was here" pins would still work. On the technical side, I settled on Expo, Supabase, Express, and react-native-maps, switching from Mapbox specifically to avoid dev build requirements; this stack also cleanly satisfied four rubric requirements (third-party API, database, frontend-backend integration, and physical phone deployment). I planned the AI-assisted versus hand-written split upfront, delegating scaffolding and JSX (Layers 1 and 3) to Cursor while writing the state machine reducers and useEffects (Layer 2) myself so I could defend the logic in the oral exam. Development followed distinct phases rather than happening all at once: scaffolding, then schema with auth and RLS, New Entry, Map, Journal with edit, Friends, and finally polish. I'm currently mid-Phase 3. Along the way, I ran into real bugs that each taught me something: an AsyncStorage native module null caused by using npm instead of npx expo install, a Supabase key rotation after accidental exposure, a username collision inside handle_new_user(), and a TypeScript discriminated union narrowing issue in the reducer.

I mainly used Claude and Cursor as my tool, also using Gemini occasionaly. I initiallty used Opus 4.7 in Claude, however it consumes tokens at a much higher rate, so I switched to Sonnet 4.6 for basic questions. I also created new chats to avoid long context windows. For Cursor I also used Sonnet 4.6 instead of auto, which was a change made the first time. I used Agentic Building, planning out the whole project and give long, sophisticated prompts for the agent, asking it to build phase by phase.

I learned to distinguish between Cursor's "command to run" suggestions and actually verifying commands work. The agent's reports described expected behavior, but real verification caught multiple disconnects (e.g., stale nodemon processes showing "clean exit" after Ctrl+C, SQL linter warnings that weren't in the prompt). Treating the agent as a collaborator whose output I verify — not as an executor I trust blindly - was the single biggest shift from how I imagined using AI.

Before taking 15-113, I only copy snippets of code from chatboxes when I use AI to write code, which makes the total process agonizing. Learning how AI can also work better with a detailed plan (very much like humans) is my most valuable find in this course. I care about my work efficiency as much as I do with quality, so Agentic Build will definitely be my takeaway. I love this App that I built for Project 3, and genuinely love everything I created in this class.
